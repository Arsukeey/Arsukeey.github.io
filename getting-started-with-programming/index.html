The Struggle

When you are getting started with programming, you may be confused with which programming language you should learn (even though you are not even sure what those are).

So I’ll give you some help. Stop trying to do that!

You are spending precious time you could be studying or programming!

Just pick whatever language you think you may like. If you have no idea of which one you should start with, I’ll give you some tips.

Reminder: Right now you are looking to learn programming logic, not the ins and outs of the language.

Programming languages beginners can use to learn

  1. Math. Seriously, this is the most important part. When you are starting to learn the logic behind the magic, math will be your best ally.

  2. Python. Simple and mature language with a syntax close to English. Good to learn.

  3. C. Old language but very, very established. Most of the modern day programming languages have their roots here, and if you think that’s not enough reasons, with C you’ll learn about how the computer does thing under the hood.

  4. Scheme. People may not agree with me on this one, but this is my blog. In my opinion, Scheme is good to learn because it’s syntax is almost non-existent. And! You have Structure and Interpretation of Computer Programs (SICP), which is one of the best Computer Science books there are, to introduce you to Scheme.

Where you can learn (resources)

I personally learned programming by using Sololearn (I’m not getting paid to write this) and I think it’s an awesome platform. If you dislike it, you can always read a book! There are lots of them. And you don’t need to pay for them, since you have zlib.

Some good books I’d recommend are:

  • SICP (as I said before)
  • The C Programming Language, 2nd Edition // After you finish this, Modern C
  • The Little Schemer
  • The Practice of Programming
  • Code Complete
  • Clean Code
  • The Pragmatic Programmer

Now, you’ll need a workspace to learn it. For hardware, mostly anything works. But everything will be easier for you if you install a Linux distribution and a text editor/IDE. Most people like to use Sublime Text or VS Code to edit text, but you can use whatever works for you. (If you care, I’ve been using neovim, and I plan to write about it on the future)

What next?

Ok, so you now know the basics of programming. Good! My sincere congratulations.

Now, learn about

  • HTML/CSS/JS. The meat and bones of the web. Since the web is the ultimate GUI application, you need to learn this.

  • Concurrency. Learn how to make asynchronous programs.

  • Git. Contribute and learn to open-source projects.

  • Java/Swift. You will use Java to build Android mobile applications, and Swift for iOS apps.

  • Functional programming. Learn Haskell, Lisp dialects and Erlang/Elixir. Seriously, you will be a better programmer, you will think more mathematically and if you get the concepts right, you will build safer applications.

  • C++. This one will teach you Object-Oriented Programming, which (sadly) is still a strong skill on the job market these days. After you are good with C++, learn Rust. “Rust is like 5 languages combined, but instead of ending up like 5 kids in a trench coat, it ends up like the power rangers” Rust has all the good features of C++ but without the boring part.

  • Networking.

  • Node.js, npm and React
