The History of Linux

September 20, 2019 by Alice Micheloni

The Complete History of Linux

Unix

To talk about Linux, we first need to talk about Unix, the operating system who inspired GNU (and that’s what made Linux useful initially).

In 1969, Unix’s development was started. It’s intent was to be used by programmers inside AT&T’s labs, and by no means it was meant to be portable.

It was officially announced outside of Bell Labs in 1973.1

GNU

GNU’s development officially started in 27th September 1983, when Richard Matthew Stallman sent this message to the net.unix-wizards newsgroup.

His plan was to make a Unix implementation from scratch, but this time, he shared it for free, since he wanted to share his software, and he couldn’t, because of copyright enforcement by software sellers.

He wanted to create a kernel, user tools, editor, compiler, etc. All from scratch and with something that would become to be known as a Free Software License, in the future (To be exact, 1989).

On March 22 of 1987, Richard Stallman (referred to as rms hereby) finished writing GCC’s first version, the GNU’s Compiler Collection. He pushed it to the MIT servers (where he studied), and hoped for the best.

By 1989, the only major missing component of GNU was the kernel. It’s development began in 1990, this time by the name of Hurd. However, Linus Torvalds, a 21-year-old Finnish student, started working on an operating system as a hobby. He got inspiration from MINIX, and when he got the simple things out of the way, he sent this message, sharing his progress on the comp.os.minix newsgroup.

In September of 1991, Torvalds finished version 0.01 of Freax (Original name of Linux, before being renamed by the then-system-administrator of the FTP server of FUNET. Freax is an portmanteau of Free + Freak + UniX).

By 1992, Linux was capable of running the X Window System, and GNU had a text editor (Emacs), a debugger (gdb), a yacc parser generator (bison), a linker (ld), an almost completed shell (Bash), a portable optimizing compiler (gcc) and TeX for text formatting.

Then, rms thought he could use Linux instead of Hurd by now, as a temporary fix (how little he knew). So he asked Torvalds to license it with the GPL, and Torvalds did so.

This kernel is (C) 1991 Linus Torvalds, but all or part of it may be redistributed provided you do the following:

  • Full source is available for free.
  • Copyright notices must be intact. Small partial excerpts may be copied without bothering with copyrights.
  • You may not distribute this for a fee, not even “handling” costs.

As Linux is a monolithic kernel, it handled with file systems and memory management altogether. This was the main focus of its criticism, since most kernels in 1991 were microkernels, who handled everything separately.

“LINUX is obsolete; LINUX is a monolithic style system, this is a giant step back into the 1970s.“`

First Linux Distributions

The very first Linux distribution was Boot-Root, made by H. J. Lu. It was called so because it was composed of two floppy disks, one named “Boot” and the other named “Root”. It contained Linux version 0.12.

The earliest actual Linux distribution is Manchester Computing Centre Interim Linux, released in February 1992.

However, the earliest Linux distribution available to the public was Yggdrasil Linux/GNU/X, released in December 1992, which was the first Linux distribution to ship with X Window System working without needing any preconfiguration (adapting itself to the hardware).

Then, on July 17, 1993 Slackware 1.0 was released by Patrick Volkerding!

almost Modern-day Linux Distributions

Slackware, together with Debian (released in December of the same year) marked a giant step in the Linux world. They both still exist to this day!

Linux 1.0 was released in March 1994, which was more of psychological step than a technological one.

The first commercial Linux distribution was SuSE Linux, based on Slackware, making Linux kind of famous in Germany.

Royalties

On August 15 of 1994, William R. Della Croce, Jr. trademarked “Linux”, and wanted 10% of all gain anyone made with the name. He didn’t even know what Linux was, he just trademarked it in an attempt to make easy money. This affected SuSE tremendously, since the company was just three college students.

Only around 1997 Torvalds managed to get the trademark lifted.

for real now Modern-day Linux Distributions

Red Hat, Inc. was founded in 1993, and it has since then brought open-source software into the enterprise world. Moreover, in 1994 Marc Ewing released Red Hat Enterprise Linux.

Canonical Ltd. was founded in March of 2004, and has brought with it the most currently popular Linux distribution, Ubuntu.

Since I’m talking about the modern Linux distributions, I’ll also talk about Arch Linux, Void Linux and NixOS, which are my favorite Linux distributions, as I write this article.

Arch Linux

Arch Linux follows the KISS (Keep It Simple, Stupid) principle by design. It tries to focus in elegance, minimalism and simplicity. It was inspired by CRUX, another minimalist distribution, that Judd Vinet started the Arch Linux project. He chose the word because of the band “arch enemy”. Vinet led the project until 1st of October of 2007, when he stepped down to name Aaron Griffin as the project’s new leader.

Relying on complex tools to manage and build your system is going to hurt the end users. […] “If you try to hide the complexity of the system, you’ll end up with a more complex system”. Layers of abstraction that serve to hide internals are never a good thing. Instead, the internals should be designed in a way such that they NEED no hiding.

  • Aaron Griffin

Void Linux

Void Linux is an independent Linux distribution that uses the X Binary Package System, and uses the runit init system.

It was created in 2008 by Juan Romero, a former NetBSD maintainer, who wanted to test out his new package manager, XBPS.

Void Linux allows you to use musl instead of the more common glibc, allowing Void to be used in embedded systems. It also is the first Linux distribution to use LibreSSL as the system crypto library by default.

NixOS, and how it changed the game

Nix, NixOS’ package manager, utilizes a purely functional approach to how software is installed. It ensures every package installed through it is in its own unique directory together with its dependencies, generated through a cryptographic hash.

This system solves the old dependency hell problem in an incredibly elegant way.

NixOS is built on top of the Nix package manager. It started in 2003 as a research project, with the Integrating Software Construction and Software Deployment2 paper Eelco Dolstra wrote.

One of its features is the declarative system configuration model, using the Nix domain functional build language. You configure networking, bootloader, kernel and so on by using said language. e.g.:

{
  boot.loader.grub.device = "/dev/sda";
  fileSystems."/".device = "/dev/sda1";
  services.sshd.enable = true;
}

Atomic Upgrades

It also uses an atomic approach to upgrading. 3 This means that if the upgrade to a new configuration is interrupted, for example, the power fails half-way through, the system will still boot fine. In other systems, the machine might end up in an inconsistent state, and may not even be able to boot anymore.
