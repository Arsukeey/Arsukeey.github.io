Programação Funcional

August 16, 2019 by Alice Micheloni

Essa é uma breve introdução à programação funcional que por acaso usa Elixir para exemplos, mas você não precisa saber Elixir para os entender, então não se preocupe.

História

Programação funcional foi “fundada” em 1958 com Lisp por John McCarthy, e ela é um paradigma de programação que tem ganhado bastante fama recentemente, já que ela faz programas serem mais legíveis e fáceis de manter, já que ela tenta combinar computações e funções matemáticas, assim causando nenhum efeito colateral.

Lisp foi fundamental para o campo do desenvolvimento de IAs, e a tecnologia provavelmente não estaria nesse estado se não fosse por Lisp.

Cálculo Lambda

A base da PF1 é Cálculo Lambda. Ela (calculo lambda) foi inventado por Alonzo Church na década de 30, e ela diz que funções devem ser representadas como caixas negras.

Por exemplo:

# Retorna o quadrado do argumento
fn x -> x * x end

Esse conceito é normalmente chamado de closures ou funções anônimas em outras linguagens é o arroz com feijão da PF.

Map, filter e reduce

Essas são famosas funções que foram introduzidas com Lisp, presentes em praticamente qualquer linguagem de programação moderna (junto com cálculo lambda), e elas aplicam transformações em arrays ou listas.

Map - aplica uma função em cada elemento de uma lista.

# Retorna [1, 4, 9]
Enum.map([1, 2, 3], fn x -> x * x end)

Filter - retorna uma lista que, de acordo com a função dada, é verdadeira

# Retorna apenas números pares, [2, 4]
Enum.filter([1, 2, 3, 4], fn x -> if rem(x, 2) == 0 do x end)

Reduce - transforma uma lista em um único valor.

# Retorna 75 (todos os elementos da lista somados)
Enum.reduce([5, 10, 15, 20, 25], fn x, acc -> x + acc end)

Imutabilidade

Um dos princípios da PF é a imutabilidade, e isso significa que o valor das variáveis em nenhum momento. Isso é importante, pois ele transforma as funções em caixas pretas previsíveis, que pegam um valor X e retornam Y. E, como efeito colateral, o desempenho é melhor, já que os valores não são copiados (pass-by-copy).

Funções de primeira classe

Isso significa que funções também são tratadas como valores, e isso também significa que elas podem retornar funções e receber funções como parâmetros, as tornando muito mais poderosas.

Funções de ordem superior

Essas são funções que:

  • ou recebem funções como parâmetros; ou
  • retornam uma função
