Programação Funcional

Essa é uma breve introdução à programação funcional que por acaso usa Elixir para exemplos, mas você não precisa saber Elixir para os entender, então não se preocupe.

História

Programação funcional foi “fundada” em 1958 com Lisp por John McCarthy, e ela é um paradigma de programação que tem ganhado bastante fama recentemente, já que ela faz programas serem mais legíveis e fáceis de manter, já que ela tenta combinar computações e funções matemáticas, assim causando nenhum efeito colateral.

Lisp foi extremamente importante para o campo do desenvolvimento de IAs, e a tecnologia provavelmente não estaria nesse estado se não fosse por Lisp.

Cálculo Lambda

A base da PF1 é Cálculo Lambda. Ela (calculo lambda) foi inventado por Alonzo Church na década de 30, e ela diz que funções devem ser representadas como caixas negras.

Por exemplo:

fn x -> x * x end # Retorna o quadrado do argumento

This concept, normally called closures or anonymous functions in other programming languages, is the core of FP.

Esse conceito é normalmente chamado de closures ou funções anônimas em outras linguagens é o arroz com feijão da FP.

Map, filter e reduce

Essas são funções famosas que vieram junto com Lisp, implementadas em praticamente qualquer linguagem de programação moderna (junto com cálculo lambda), e elas aplicam transformações em arrays ou listas.

Map - aplica uma função em cada elemento de uma lista.

Enum.map([1, 2, 3], fn x -> x * x end) # retorna [1, 4, 9]

Filter - retorna uma lista que, de acordo com a função dada, é verdadeira

Enum.filter([1, 2, 3, 4], fn x -> if rem(x, 2) == 0 do x end) # Retorna apenas números pares, [2, 4]

Reduce - transforma uma lista em um único valor.

Enum.reduce([5, 10, 15, 20, 25], fn x, acc -> x + acc end) # Retorna 75 (todos os elementos da lista somados)

Imutabilidade

Um dos princípios da PF é a imutabilidade, e isso significa que o valor das variáveis não muda. Isso é importante pois ele transforma as funções em caixas pretas previsíveis, que pegam um valor X e retornam Y. E, como um efeito colateral, a performance é melhor, já que os valores não são copiados por aí.

Funções de classe alta

Isso significa que funções também são tratadas como valores e elas podem ser usadas como valores; E isso também significa que elas podem retornar funções e receber funções como parâmetros, as tornando muito mais poderosas.

Higher order functions

Essas são funções que: - ou recebem funções como parâmetros, ou - retornam uma função

Combine all that we just talked about, and you get the powerful mix that FP is! Thank you for reading.
